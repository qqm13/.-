Отлично, вот упрощённые задачи, которые понятны любому участнику команды, включая новичков. Формат: «Сделать [что] для [кого]».

---

Задачи по входу в систему и паролю

1. Сделать страницу входа с формой «логин/пароль» для всех пользователей.
2. Сделать кнопку «Забыли пароль?», которая отправляет ссылку для сброса на почту.
3. Сделать так, чтобы после входа пользователь попадал в свой личный кабинет.

Задачи для студента

1. Сделать страницу «Моё расписание», где студент видит свои занятия на неделю.
2. Сделать страницу «Моя посещаемость» — таблицу с датами, предметами и статусами (был/не был).
3. В таблице посещаемости сделать кнопку «Объяснить отсутствие» напротив каждого пропуска.
4. Сделать всплывающее окно для объяснения пропуска: поле для текста причины и кнопка «Загрузить справку».

Задачи для преподавателя

1. Сделать для преподавателя страницу «Мои занятия сегодня» со списком пар.
2. Сделать кнопку «Отметить посещаемость» для каждого занятия в списке.
3. Сделать страницу отметки: список студентов группы с переключателями «присутствовал», «отсутствовал», «опоздал» и кнопка «Сохранить».
4. Сделать страницу «Статистика посещаемости» по моим предметам, где видно, кто сколько пропустил.
5. В статистике сделать так, чтобы можно было кликнуть на пропуск и увидеть причину от студента (если он её указал).

Задачи для администратора (отчётность)

1. Сделать страницу «Отчёты» с тремя вкладками: «По студенту», «По группе», «По курсу».
2. На вкладке «По студенту» сделать поиск студента по ФИО и показ всей его посещаемости.
3. На вкладке «По группе» сделать выбор группы из списка и показ сводной таблицы посещаемости для всех её студентов.
4. На вкладке «По курсу» сделать выбор курса (1, 2, 3...) и показ общей статистики по нему.
5. Над каждым отчётом сделать кнопку «Скачать в Excel», которая выгружает таблицу в файл.
6. На странице отчётов сделать фильтры: «Выбрать период», «Выбрать предмет», «Только пропуски».

Общие задачи по профилю

1. Сделать в личном кабинете раздел «Мой профиль», где можно посмотреть и изменить своё ФИО и email.
2. Сделать шапку сайта с именем пользователя и кнопкой «Выйти».

Технические задачи «под капотом»

1. Создать базу данных с таблицами: Пользователи, Группы, Расписание, Посещаемость.
2. Написать на бэкенде код, который проверяет логин и пароль и выдаёт токен.
3. Настроить права доступа: чтобы студент не мог зайти на страницу отчётов, а администратор — на страницу отметки посещаемости.
4. Сделать API-метод для загрузки файла (справки) на сервер.
5. Написать скрипт, который формирует Excel-файл из данных для отчёта.

---

Эти задачи можно сразу раздавать разработчикам, тестировщикам и дизайнерам. Каждая задача — это конкретное, осязаемое действие.
